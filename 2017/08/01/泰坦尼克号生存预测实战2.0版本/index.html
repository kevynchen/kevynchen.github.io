泰坦尼克号生存预测实战2.0版本

发表于 2017-08-01

这个2.0版本主要是对于特征工程进行了一些优化，比如对title头衔进行了重新划分,因为参考别人的经验title时最重要的。新增加了一个FamilyID特征，以及把最后的randomForest预测函数修改为了cforest函数，果然最后的预测排名分数有所提高，但是也只是达到了前9%而已。

在不断地修改代码，上传，将之前的好几个变量比如title,Fsize,Fare进行了一些处理后发现都不如意，分数反而下降了，才明白特征工程选取的重要性，如果你选取的特征不合适，或者特征太多，比如我之前提取出来title后，并未做任何的处理，得分0.789，后面经过合适的处理，将一些少部分的头衔给归类，才算是把title给处理好了。

总而言之，就是特征选取非常重要，不断地尝试，探索，最后找到比较合适的特征。也只有选取的特征合适了，我们的模型才能够无限地逼近我们想要的那个预测的概率，而不是模型大于一切，在这个过程中，我看kaggle以及社群会员的一些实践，发现使用cforest的得分一般会比使用randomForest的高一些，可能就是cforest能够无限逼近那个概率吧。

提取title的时候出了点问题，弄了好久不知道什么原因，结果发现是提取出来的title前面多了一个空格，从而导致对title部分重新赋值失败。

最终的解决办法是在stackoverflow上面询问的答案，结果不到30分钟就有人回答我这个问题了，真的是及时雨啊，非常地感谢。

接下来就是具体的实践情况。

1. 数据处理目标

通过官方给的数据train 数据集来预测test数据集的生存状况，survived = 0表示死亡，survived = 1表示活下来了。
这个数据总共有1309个观测量，12个变量值。我们的目的就是要利用各变量与survived之间的关系来得出我们的结论。

2. 数据预处理

载入我们后面处理数据需要用到的包，并且为了后面处理数据方便，我们将train和test合并到一个数据框里面。

然后我们查看数据，得到下面的各数据变量类型的一个预览。

从这里我们知道有些变量缺失值比较多，所以接下来就是对缺失值进行填补，利用中位数和平均值的方法。下面就进行具体的变量的处理。

3. 特征工程

从乘客的名字我们可以看出来，乘客的名字类似这样Braund, Mr. Owen Harris，一个逗号，然后空格再加一个头衔，后面再空格；这里我们可以利用字符串分解的正则表达式将头衔从名字中提取出来。这里title前面多了一个空格，利用trimws()函数给修剪掉。

到这里我们可以很清楚得看到各头衔的人数有多少了。

接下来就是要把名字的姓给拆分出来，并查看家庭成员的大小对生存的影响，家庭成员指的是乘客和乘客的亲戚兄弟姐妹，父母孩子。

这里的FamilyID参考了别人的想法，考虑到外国人中姓氏一般很少重复，而名字则很有可能重复，而姓氏相同的可能为一家人，所以可能同时遇难或者获救，这仅仅是一种考虑方法，所以把这种姓氏比较少的提取出来，并且命名为small,其余则不变。

接下来就是看乘客登船港口与生存之间到底存在着什么关系，从下面的ggplot2图中我们可以看到从S港口登船的第一阶层的乘客的花费与80美元刚好相符，所以我们可以预测缺失了登船港口信息的乘客的地址为“C”港口。

然后接下来就是探索第三阶层，从S港口登船的人的平均船票的费用。用均值的方法。查询出来得到其平均费用为8美元，然后填补缺失的费用的数值。

对于船票的票号一开始并没有进行处理，现在进行处理。把票号分为共享的和单独使用的，共享票号的有可能是一家人，可能同时遇害或者获救，而单独的我们画出ggplot图看看他们的关系是怎么样的，结果发现共享票号的生存率达到了50%以上，而单独票号的遇害率非常大。

最后就是估测其余的缺失值，得到这样的一个表。

同时，根据我们观看泰坦尼克号的经验可以得知，妇女和小孩是先走的，我们用ggplot2 制作出相关的图形，看看是不是这样的情况，从下面的图中我们可以看到小孩和妇女的生存率都比较高，而男性则非常地低，符合我们的常识；在这里，我们把年龄小于18的称之为小孩，而大于18的则为成年人；对于母亲来说，满足母亲的条件有以下几个：性别为女，有至少一个孩子，头衔不能为“Miss”，年龄大于18岁，其他的则为非母亲，在这里，非母亲我们不考虑男人等等，只考虑母亲这一个因素。

4. 建立模型，对test数据集进行预测

到这里，我们基本把所有的缺失值都已经填补完毕了，接下来就是利用随机漫步算法建立模型，然后对test数据集进行预测。

最后将预测的结果上传得到了kaggle官网，得分0.80861，排名673，虽然不是很高，但也算是基本摸清了特征选择的一个大概的思路，接下来要多多参加这样的比赛了。同时这也只是public部分的成绩，不一定代表在private部分，我们的模型也能够跑得很好，做出高的预测，毕竟模型就是要未卜先知，而不是对已有的数据进行预测地多好。
